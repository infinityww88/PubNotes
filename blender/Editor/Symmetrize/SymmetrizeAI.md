非常好 👍，你问的 **Blender 骨骼的 “Symmetrize”** 功能是骨骼编辑中非常常用、非常实用的一个自动对称工具。
下面我帮你详细讲清楚它的作用、使用步骤、命名规则以及常见问题 👇

---

## 🧩 一、Symmetrize 是什么？

**Symmetrize（对称化）** 是在 **骨骼编辑模式（Edit Mode）** 下使用的命令，作用是：

> ✳️ **自动根据一侧的骨骼，创建镜像的另一侧骨骼，并保持命名与方向一致。**

简单说就是：

* 你建好一边的骨架（比如左臂）；
* 一键生成右边（右臂）；
* 自动命名成对应的 `.L` 和 `.R`；
* 自动翻转方向与位置。

---

## 🦴 二、Symmetrize 的使用步骤

### **步骤 1：建好一侧的骨骼**

例如在 X 轴的左边建好一条手臂骨架：

```
upper_arm.L
forearm.L
hand.L
```

（注意命名里的 `.L`）

### **步骤 2：进入编辑模式**

选中 Armature 对象 → 按 `Tab` 进入 **Edit Mode**。

### **步骤 3：选择要镜像的骨骼**

选中你想镜像的那一侧（通常是左侧）的骨骼。
（如果不选，Blender 会自动尝试对整个骨架执行。）

### **步骤 4：执行命令**

使用以下任一方式：

* 菜单：`Armature → Symmetrize`
* 快捷键：`Ctrl + Shift + M`

---

## ⚙️ 三、命名规则（非常关键）

Symmetrize 依靠骨骼名字识别哪一侧是左、哪一侧是右。
必须遵守以下命名后缀之一：

| 左侧骨骼后缀  | 镜像后缀     | 示例                         |
| ------- | -------- | -------------------------- |
| `.L`    | `.R`     | `arm.L` ↔ `arm.R`          |
| `_L`    | `_R`     | `leg_L` ↔ `leg_R`          |
| `.Left` | `.Right` | `eye.Left` ↔ `eye.Right`   |
| `_left` | `_right` | `wing_left` ↔ `wing_right` |

💡 **小贴士：**

* `.L` / `.R` 是最常用、最标准的命名方式。
* Symmetrize 会根据 X 轴反转位置：左边（X < 0）→ 右边（X > 0）。

---

## 🧩 四、Symmetrize 的原理

* 对选中骨骼执行镜像变换（关于 X=0 平面反射）。
* 自动创建新骨骼（不是原骨骼的实例）。
* 保留父子层级结构。
* 自动更新名字后缀（`.L` → `.R`）。

---

## 🧠 五、使用注意事项

1. **需要在 Edit Mode 下使用**，Pose Mode 下无效。
2. **中心轴必须在 X=0 对称位置**。

   * 如果骨架整体偏离原点，对称会不准确。
3. **骨骼命名要符合规范**，否则不会自动配对。
4. **重复执行不会报错**，但可能覆盖原来的对称骨。

---

## ✅ 六、常见用法举例

| 场景             | 操作                              | 效果            |
| -------------- | ------------------------------- | ------------- |
| 建好一边的手臂骨骼      | 选中 `.L` 骨骼 → `Ctrl + Shift + M` | 自动生成 `.R` 手臂骨 |
| 修改左侧手臂形状后想同步右侧 | 只留左侧骨骼 → Symmetrize             | 右侧更新为镜像版本     |
| 修复丢失的对称骨       | 选中左/右任意侧 → Symmetrize           | 重新补全缺失的一侧     |

| **命名要求**     | `.L` / `.R`（或 `_L` / `_R` 等） |
