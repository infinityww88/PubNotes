对于 articulationbody，使用 FixedJoint 将两个刚体固定在一起，与将它们的 collider 组合在一起作为一个刚体的差别大吗？
##  一、两种方式的定义

| 方式                                      | 说明                                                                                                |
| --------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **多个 ArticulationBody + FixedJoint 连接** | 用 `ArticulationBody` 组件分别放在每个物体上，用 `jointType = FixedJoint` 或 `jointType = Locked` 约束它们，保持相对刚性连接。 |
| **单个 ArticulationBody + 多个 Collider**   | 只有一个 `ArticulationBody`，多个形状（Collider）作为该物体的碰撞体组成一个整体刚体。                                          |

## 二、物理模拟层面的根本区别

| 特性           | FixedJoint 连接多个 AB                                                               | 多个 Collider 组成一个 AB                       |
| ------------ | -------------------------------------------------------------------------------- | ----------------------------------------- |
| **自由度与约束计算** | 每个 joint 都会引入一组约束方程（锁定相对位置与旋转），Unity 物理引擎（PhysX Articulation Solver）需要在每帧解算这些约束。 | 无约束计算，系统直接认为这些 Collider 共享一个刚体的惯性与速度状态。   |
| **数值稳定性**    | 受解算误差影响，FixedJoint 可能出现微小“抖动”或数值漂移。<br>连接层级越多，误差累计越明显。                           | 绝对稳定，不会分离或抖动，因为它们属于同一个物理体。                |
| **性能开销**     | 每个 joint 都是一个约束方程，会显著增加计算量，尤其是多层结构。                                              | 无额外约束开销，性能最佳。                             |
| **惯性和质心计算**  | 每个刚体独立有 mass 与 inertia tensor，joint 连接时物理引擎实时计算整个系统的综合惯性。                        | 所有 Collider 的 mass、惯性矩等直接合并计算，数学上更简单也更准确。 |
| **受力与扭矩传递**  | 力通过 joint 传递，会有延迟和非线性（取决于 solver 精度）。                                            | 力直接作用在一个整体上，传递完全一致。                       |
| **碰撞响应**     | 各自独立检测碰撞，响应通过 joint 传递。可能导致“局部晃动”。                                               | 整体响应，无接缝、无晃动。                             |

## 三、何时用哪种方式

| 使用场景                    | 推荐方式                                    | 原因                |
| ----------------------- | --------------------------------------- | ----------------- |
| 🚗 单个刚体但形状复杂（如车身）       | ✅ **一个 ArticulationBody + 多个 Collider** | 更稳定、更高效。          |
| 🤖 有机械运动或可动关节（如机器人手臂）   | ✅ **多个 ArticulationBody + 关节连接**        | 需要独立自由度。          |
| ⚙️ 想“看起来连在一起”但要能检测受力、断裂 | ✅ **FixedJoint**                        | 可在受力大时断开 joint。   |
| 🔩 完全刚性且永不分离的组合         | ❌ 不建议用 FixedJoint                       | 合并成一个刚体更好（更快、更稳）。 |

## 四、实践建议

1. **避免“假刚体”组合**
   若两个物体永远不会相对移动，就不要用 FixedJoint，而应直接用多个 Collider 组成一个 ArticulationBody。

2. **减少 joint 数量**
   ArticulationBody 的 joint 解算非常昂贵（比传统 Rigidbody Joint 贵 5–10 倍），层级越深，性能越差。

3. **在机械链中，尽量使用最少的自由度**
   比如一个 Revolute 关节就足够时，不要用 Spherical + 限制，因为那会多解算两个自由度。

### 总结一句话：

> 🔹 **FixedJoint 只是“模拟”刚性连接**，仍然有误差和性能成本；
> 🔹 **多个 Collider 合成一个刚体才是真正的刚性一体**，性能和精度都更优。
